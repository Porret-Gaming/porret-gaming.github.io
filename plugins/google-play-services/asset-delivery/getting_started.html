Getting Started

  1. Purchase the Plugin

    2. Purchase the plugin on the Official Unreal Engine Marketplace (Disabled untill Go-Live).

  2. Install the Plugin

    3. In the Epic Games Launcher, the plugin will appear in your Library.
    Go to the Library tab and find the plugin and add it to the engine.

  3. Enable Play Services Play Asset Delivery (PAD) Plugin

    4. In Unreal Engine, the plugin will be available in the plugins section.
    Go to Settings -> Plugins -> Search for "Play Services Play Asset Delivery (PAD)" and ensure it is enabled.

  4. Disable Unreal Engine's built-in Google PAD plugin

    5. In Unreal Engine, the plugin will be available in the plugins section.
    Go to Settings -> Plugins -> Search for "Google PAD" and ensure it is disabled.

  5. Enable Generate AAB Bundle

    6. You should already be doing this, but just in case:
    Go to Project Settings -> Platforms ->Android and Enable Generate bundle (AAB).

  6. Disable Package Game Data Inside APK

    7. Go to Project Settings -> Platforms ->Android and Disable Package Game Data Inside APK?.

  7. Disable OBB Patching

    8. Go to Project Settings -> Platforms ->Android and Disable Allow patch OBB file.

  8. Enable Generate Chunks

    9. Go to Project Settings -> Project ->Packaging and Enable Generate Chunks.

  9. Verify

    10. When packaging your project, the plugin will check that you have done the above correctly.
    Any misconfigurations will be logged as warnings and inform you what is wrong and consequently how to fix it.

    • Invalid Configuration Log Output
    • Correct Configuration Log Output
  10. Upload to Google Play Store

    11. When uploading to the Google Play Store, simply upload your aab file as usual and it will accept it without warning, even when your application size is well over 200MB.
